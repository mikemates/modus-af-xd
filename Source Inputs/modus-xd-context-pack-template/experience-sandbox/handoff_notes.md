## How to Use This Sandbox

Use the sandbox to understand:

- intended product flow
- route and navigation structure
- interaction behavior
- state coverage
- content direction
- component assumptions
- representative data
- experience acceptance expectations

Do not use the sandbox as:

- production architecture
- final code quality standard
- source of authentication, authorization, or API implementation
- replacement for backlog acceptance criteria
