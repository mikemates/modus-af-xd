# Agent Review Checklists

## Full XD Pack Review

- [ ] Mode is declared and rationale is clear.
- [ ] Experience intent references product intent and does not duplicate it.
- [ ] Users, tasks, and roles are specific.
- [ ] Hero flows and edge cases are documented.
- [ ] IA, routes, object model, and terminology are clear.
- [ ] Sandbox manifest is current if a sandbox exists.
- [ ] Interaction states are explicit.
- [ ] Components and patterns are named.
- [ ] Content rules are defined where needed.
- [ ] Accessibility and usability expectations are stated.
- [ ] Experience acceptance gates are clear.
- [ ] Measurement signals and instrumentation are defined.
- [ ] AI experience patterns are defined if relevant.

## Story Review

- [ ] Specific role.
- [ ] Real outcome.
- [ ] Experience reference.
- [ ] Flow ID and route ID.
- [ ] State coverage.
- [ ] Component guidance.
- [ ] Copy source.
- [ ] Accessibility expectation.
- [ ] Instrumentation event if needed.
- [ ] Observable ACs.
- [ ] Open questions visible.

## Sandbox Review

- [ ] Core flows represented.
- [ ] Major routes included.
- [ ] States visible or documented.
- [ ] Mock data sufficient.
- [ ] Component usage realistic.
- [ ] Engineering concerns captured.
- [ ] Reusable/reference-only elements marked.
- [ ] Backlog mapping complete.
- [ ] Drift log initialized.
