# Modus AF — Experience Design

Experience Design's contribution to **ModusAF** (the Modus Agentic Framework): the durable, structured context that lets humans and AI agents move from product intent to working software with less ambiguity and rework.

This work is an **extension of the Product Leadership context pack**, not a replacement. PL owns the product spine (business intent, value, prioritization, adoption). Experience Design adds the experience implementation layer — flows, IA, interaction behavior, content, accessibility, acceptance, and a coded Experience Sandbox.

## What's in here

- **`modus-xd-context-pack-refined/`** — the XD context pack: a reusable, lean template set (27 files) for authoring experience context per initiative. Start with its `README.md`, then `GETTING_STARTED.md`.
- **`modus-experience-standards.md`** — the Modus-wide house defaults (accessibility, interaction states, responsiveness, content, components). Project packs inherit these and record only their deltas.
- **`Source Inputs/`** — the original baseline: the Product Leadership pack, the first-draft XD pack, and the planning overview. Kept untouched as the reference point the refined pack was built from.
- **`PL-sync-discussion-brief.md`** — working brief for the PL collaboration session.

## Where to start

1. `modus-xd-context-pack-refined/GETTING_STARTED.md` — how to populate a pack, step by step.
2. `modus-xd-context-pack-refined/README.md` — the reference (file list, purposes, mode table).
3. `modus-xd-context-pack-refined/PL_INTEGRATION.md` — how the XD pack wires into the PL pack.
4. `modus-experience-standards.md` — the defaults every project inherits.
